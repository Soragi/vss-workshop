  When nims is enabled, the NIM Operator exposes ClusterIP services matching the
  NIMService metadata.name (same slugs as dev-profile-base nims charts):

  1. Nemotron Nano 9B v2 (LLM):  nvidia-nemotron-nano-9b-v2:8000
  2. Cosmos3 Reasoner (VLM):     nvidia-cosmos3-reasoner:8000  (when Cosmos3 subchart is enabled)
  3. Eval / judge LLM uses the same endpoint as (1).

=====================================================================
  QUICK INSTALL
=====================================================================

  helm upgrade --install vss-search ./dev-profile-search \
    -n default \
    --set global.externalHost=vss-search.<NODE_IP>.nip.io \
    --set global.ngcApiKey=<NGC_API_KEY> \
    --set nims.global.ngcApiKey=<NGC_API_KEY> \
    --wait=false

  Critic agent and Cosmos3 NIM are enabled by default.
  To disable: --set global.enableCritic=false,nims.cosmos3.enabled=false

  Both bare IP and hostname formats are accepted for global.externalHost:
    --set global.externalHost=<NODE_IP>                          (bare IP)
    --set global.externalHost=vss-search.<NODE_IP>.nip.io        (hostname)
  Ingress hosts are auto-constructed using nip.io in either case.
  Override with --set ingress.hosts.main=custom.domain.com if needed.
